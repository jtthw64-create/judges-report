# 40 · 검증 방법론

모든 세션은 이 기준으로 정제·식별한다. 목적: **가짜 없이, 근거를 명시하며** 서지를 확정.

## 1. 이 xlsx가 만들어진 계보 (오류가 생기는 지점)
```
주석서 3권 PDF (Sasson AYB · Smith Hermeneia · Groß Richter)의 참고문헌
   │ ① OCR 텍스트 추출
   ▼
extracted/bib_sasson.md, bib_smith.md          ← 1차 소스(비교적 깨끗)
   │ ② 내 드라이브와 "성씨+연도" 대조 → 미보유 필터  ★여기서 저자 밀림 다수 발생
   ▼
extracted/MISSING_{Sasson,Smith,Groß}.md
   │ ③ 인물/주제 분류
   ▼
Judges_missing_by_character.xlsx               ← 최종(오류 누적)
```
- **주의:** Groß분은 `bib_gross.md`가 없고 `MISSING_Groß_*.md`만 있음(원본 대조 소스가 약함 → 웹 의존도 ↑).

## 2. 오류의 두 층
- **(A) OCR 글자 깨짐:** `Ftnsham`→Fensham, `cglon`→Eglon, `vr`→VT, `ll`→II, `Gro8`→Groß, `254+`(쪽수 잘림).
- **(B) 저자–제목 정렬 밀림(off-by-one):** 항목 경계 오파싱으로 저자명이 옆 항목 것으로 밀림.
  - 실증: `Ethically Cultured…`는 bib_sasson.md에 **Sasson 2009**로 정확하나, MISSING·xlsx엔 **Hurowitz 2009**로 틀어짐(Hurowitz는 원본의 다른 항목).
  - 실증: Groß 목록의 `Smith 1942` 행 = 실제 **Soggin 1989** "Ehud und Eglon" VT 39.
- ⇒ **xlsx의 author·year 필드는 신뢰 불가**. 가장 안정적인 단서는 **title**.

## 3. 확정 절차 (항목마다)
1. **원본 대조 우선:** `extracted/bib_sasson.md`·`bib_smith.md`(+ MISSING_*.md)에서 title로 원항목을 찾아 저자·연도·서지를 1차 복원. (Groß분은 원본 약함 → 3번 비중 ↑)
2. **외부 서지 DB 확정:** title을 키로 저널사/DOI/WorldCat/대학 리포지토리에서 실재·저자·권·쪽 확인.
3. **교차 일치 판정:** 원본과 외부 DB가 일치하면 확정. 불일치·미확인은 등급을 낮추거나 UNRESOLVED.
> "파일(PDF) 미확보"와 "서지정보 미상"은 다르다. 판정 대상은 자료 내용이 아니라 **xlsx 서지 표기의 정오**다.

## 4. 신뢰등급 (confidence) — CSV·대시보드 공통
| 등급 | 기준 | 조치 |
|---|---|---|
| **A** | 저널/DOI/출판사 **1차 페이지** 또는 원본 bib에서 저자·제목·서지 **직접 확인** | 그대로 확보 |
| **B** | 신뢰할 **2차 요약**(검색엔진 요약, WorldCat 스니펫) 일치. 1차 미열람 | 확보하되 수령 시 대조 |
| **C** | **추정**(간접·부분 근거, 연도 등 불일치 잔존) | 수령 시 **원문 최종 확인 필수** |
| **UNRESOLVED** | title로도 식별 실패(OCR 병합 등) | `UNRESOLVED.csv` 격리, 원본 참고문헌 대조 필요 |

## 5. 상태(status)와 등급의 관계
- `RESOLVED` = 실재·서지 확정(A/B/C 등급 부여). `UNRESOLVED` = 미확정.
- download_queue 편입 = RESOLVED & not HELD_ALREADY. **C등급은 대시보드 ⚠로 표시**해 수령 시 확인 유도.

## 6. 표기 규칙 (warn vs info)
- **⚠ warn:** 받기 전 확인할 데이터 이슈 — 저자정정·쪽수정정·연도불일치·확보난이도.
- **ℹ info:** 참고 메모(경고 아님) — 중복병합·OCR 경미정정·FS 수록 등.
- 완료된 처리 이력(예: "PEQ 바운더리 추가")은 warn/info 어디에도 경고로 넣지 않는다.

## 7. 가짜 방지 체크리스트 (항목별)
- [ ] title로 **bib 원본** 대조했는가 (Sasson/Smith)
- [ ] title로 **외부 DB** 실재·저자 확인했는가
- [ ] author·year가 원본/DB와 어긋나면 정정하고 근거를 notes에 남겼는가
- [ ] 확신 강도에 맞는 **confidence(A/B/C)** 를 부여했는가
- [ ] 식별 실패는 지어내지 않고 **UNRESOLVED**로 보냈는가

## 8. 원본 폴더 보유자료 대조 (HELD_ALREADY 판정) — 2026-07-16 신설

정제 대상이 **이미 원본 폴더에 확보돼 있는지** 반드시 대조한다. 이미 있는 자료를 다시 받게 하면 안 되기 때문.

### 8-1. 대조 대상
- 원본 폴더 **전체 하위 재귀**: `../5 Book 3 Judges Resources/` 아래 모든 폴더
  (`0 Judges Commentaries`, `0 Judges general`, `1 Judg *`, `2 인물별`, `3 주제별`, `4 Joshua`, `5 epubs`, `Nuri added` 등)
- 대상 확장자: `.pdf`, `.epub`, `.docx`, `.doc` (2026-07-16 기준 604개 파일)
- **읽기 전용.** 원본은 절대 수정하지 않는다.

### 8-2. 매칭 절차 (3단계)
1. **1차 — 성씨 + 연도**
   - 저자 성씨(첫 단어)와 4자리 연도가 파일명에 모두 있는지.
   - ⚠️ **함정**: 성씨 매칭은 반드시 **단어 경계**로 할 것.
     - 부분 문자열 매칭 시 `May` ⊂ `Gillmayr-Bucher` 같은 오탐 발생(실제로 겪음).
     - 또한 저자명 토큰화에 **최소 길이 제한(4자 등)을 걸지 말 것** — `Kim`·`Bal`·`Cox`·`Nel` 같은 3글자 성이 통째로 누락됨(실제로 겪음).
   - 연도는 파일명의 숫자열에 포함되는지로 확인(`./2 Samson/Kim 1993 …` → digits `21993`에 `1993` 포함).

2. **2차 — 제목 토큰 겹침**
   - 후보 파일과 xlsx 제목의 4자 이상 단어 토큰 교집합 개수(score)를 센다.
   - 불용어 제외: `judges, judg, book, study, studies, the, and, from, with, essay, essays, vol, article, pdf, epub`
   - `score >= 1`이면 강한 후보.

3. **3차 — 사람 눈 검증 (필수)**
   - `score == 0`인 후보(성씨+연도만 일치)는 **반드시 제목을 눈으로 대조**한다.
     - 파일명이 축약형이라 겹침이 0이어도 실제 동일 문헌인 경우가 있다.
       예) xlsx `“Hidden Polemic in the Conquest of Dan”` ↔ 파일 `Amit-HiddenPolemicConquest-1990.pdf` (붙여쓰기라 토큰 불일치, 실제로는 동일)
     - 반대로 **동일 저자·동일 연도의 다른 문헌**일 수 있다.
       예) Naʾaman 2005 `Canaan in the Second Millennium` ≠ 파일 `Naaman 2005 Danite Campaign Northward` → **제외해야 함**(실제로 걸러낸 사례)
   - 자동 점수만 믿고 일괄 반영하지 말 것.

### 8-3. 반영 방법
1. **xlsx 사본** (`judges report/Judges_missing_by_character.xlsx` — 원본 아님)
   - 각 시트 마지막 열에 `확보상태` 헤더 추가, 해당 행에 **`완료`** 기입.
   - 카테고리 시트와 `전체 All` 시트 **양쪽 모두** 반영.
   - `openpyxl`로 `data_only=False` 로드해 서식 보존.
2. **`worklist/download_queue.csv`**
   - `status`: `QUEUED` → **`HELD_ALREADY`**
   - `notes` 끝에 `[확보완료: <원본폴더 상대경로>]` append
   - 17컬럼 순서 유지, 콤마 포함 필드는 큰따옴표.
3. **대시보드**: `HELD_ALREADY` 시각 구분은 **Codex 담당**(`70_Codex_UI_요청.md` WO-001).
4. `python3 build_dashboard.py` 재생성 → 커밋 → push.

### 8-4. 2026-07-16 1차 대조 결과

### 8-5. 2026-07-17 재점검 — 누락 11건 추가 발견
사용자 지적("확보완료 누락이 많다")으로 `download_queue.csv`의 QUEUED 197건 전체를 성씨+연도+**제목토큰겹침** 스크립트로 재대조. **11건 추가 확정**(모두 K/J/I/H/G/F/E 등 **HELD_ALREADY 절차 신설(2026-07-16) 이전에 정제된 청크**에서 누락됨 — 8-4의 "xlsx 전체 스캔"이 실제로는 이 항목들을 잡지 못했던 것으로 확인):
`E-01-003`(Bluedorn 2001), `E-01-012`(Emerton 1976), `E-01-019`(Haag 1967), `E-01-030`(Malamat 2004), `E-01-042`(Assis 2005 Self-Interest), `F-01-011`(Fuchs 1989), `F-01-027`(Römer 1998), `G-01-016`(Noll 1995, `Nuri added` 하위폴더), `I-01-001`(Sasson 2009), `I-01-004`(Soggin 1989), `J-01-001`(Fensham 1961).

**교훈**:
- 성씨+연도만으로는 오탐(같은 저자·같은 해의 다른 논문)과 누락이 둘 다 생긴다. **제목 토큰 겹침 점수(score)까지 반드시 계산**하고, score=0 후보라도 원 정제 세션에서 "저자정정" 특이사항으로 기록된 항목(예: I-01-004 Soggin 1989, I-01-001 Sasson 2009는 각각 세션3·서브에이전트가 저자 오기를 잡아낸 유명 사례)은 **재정정 후 반드시 held-check도 다시 돌릴 것** — 저자가 바뀌면 매칭 결과도 바뀐다.
- `Nuri added` 같은 하위폴더가 카테고리 폴더 안에 또 있을 수 있으니 **재귀 스캔 필수**(`os.walk` 또는 `find`에 심볼릭 링크 팔로우 옵션 필요 — 원본폴더 경로 자체가 심볼릭 링크라 `find -L` 또는 `os.walk(..., followlinks=True)` 안 쓰면 0건으로 나온다, 실제로 겪음).
- **앞으로 원칙**: 정제 완료 여부와 무관하게, `download_queue.csv`가 갱신될 때마다(신규 청크 편입 시) 전체 QUEUED 행에 대해 이 재점검 스크립트를 다시 돌린다. 한 번 통과했다고 끝난 게 아니다.

## 9. 서브에이전트 위임 시 필수 고지 사항 — 2026-07-17 신설

대형 청크는 Sonnet 서브에이전트 3분할 병렬 위임이 표준이다(B·A 배치에서 9서브에이전트 동시 실행이 안정적으로 작동). 다만 위임 프롬프트에 **아래를 반드시 명시**할 것.

### 9-1. 원본 bib 경로를 정확히 줄 것 (★실패 사례)
- **정확한 경로**: `5 Book 3 Judges Resources/judges-index/extracted/`
  (`bib_sasson.md` · `bib_smith.md` · `MISSING_Sasson_…md` · `MISSING_Smith_…md` · `MISSING_Groß_…md`)
- **작업 폴더(`judges report/`) 하위가 아니다.** 원본 폴더(읽기전용) 안에 있다.
- **실제로 겪음(2026-07-17)**: 커맨더가 `{작업폴더}/judges-index/extracted/`로 잘못 안내 → A-01c 담당 에이전트가 파일을 못 찾자 **질문 없이 1단계(원본 대조)를 통째로 생략**하고 외부 DB만으로 50행을 확정해버렸다. 올바른 경로로 재작업시키자 **A등급 29→44**로 뛰었고, UNRESOLVED였던 R121이 해결(Bekkum→Benz 1972)됐으며, 외부 DB가 준 **틀린 어순·누락 다이어크리틱** 9건이 추가로 잡혔다.
- **교훈**: 1단계는 장식이 아니다. 외부 DB만으로 확정하면 그럴듯하지만 틀린 서지가 A등급을 받는다. 서브에이전트가 "원본을 못 찾았다"고 보고하면 **결과를 채택하지 말고 경로를 고쳐 재작업**시킬 것.

### 9-2. 중복은 "표시"가 아니라 "병합"까지 지시할 것
서브에이전트들은 중복을 정확히 **인지**하고도 양쪽 행을 모두 남기는 경향이 있다(notes에 상호 참조만 기재). 이전 청크(D-01 "62→58 고유" 등)의 관례는 **고유화(병합)**이므로, 위임 시 "중복 발견 시 1행으로 병합하고 cited_by를 통합하라"고 명시하거나 커맨더가 통합 단계에서 병합할 것.
- **병합 시 주의(★실제로 겪음)**: 버리는 행에만 `[확보완료: 경로]`가 있을 수 있다. 2026-07-17 Bal 1987 병합에서 경로를 가진 R81 행을 버려 경로가 유실됐다(복구함). **병합 후 반드시 HELD_ALREADY 전건에 `[확보완료:` 가 남아있는지 전수 검증**할 것.
- **병합하면 안 되는 함정**: 제목이 같아도 별개 자료일 수 있다. 예) IDB 본편(Buttrick 1962, 4vols) vs IDB **보유판**(Crim 1976 Supplementary), Dossin *Correspondance de Šamši-Addu* **ARM 1권**(1950) vs **ARM 4권**(1951). 반대로 초판/재간(Brichto 1963/1968, Cross-Freedman 1975/1997)은 동일 저작 → 수령은 1건이면 충분.

### 9-3. author 필드 오염 점검
서브에이전트가 author 필드에 서지 메타(`Barron, [PhD diss.]` 등)를 섞어 넣는 사례가 있었다. 통합 시 author 필드에 `[`·`diss`·소문자 시작 등이 있는지 정규식으로 훑을 것.

### 9-4. 단행본·챕터 항목의 boundary 누락 (2026-07-17 발견)
`OT-검색바운더리.md`의 저널·시리즈 표는 논문(저널) 위주로 체크되고, **단행본/챕터 항목의 `journal_series` 필드**(예: `"(book; Forschungen zum Alten Testament 51; ...)"`, `"in: Hg., ... (FAT 32; ...)"`처럼 서두가 `(book;` 또는 `in:`으로 시작하는 경우)는 시리즈명이 문장 중간에 묻혀 있어서 boundary 판정 시 놓치기 쉽다. 실제로 FAT(Forschungen zum Alten Testament, 통독 시리즈) 소속 단행본 7건이 `off-list`로 잘못 분류된 채 A-01~A-03에 편입돼 있다가 사용자 지적으로 발견·정정됨.
**교훈**: boundary 판정 시 `journal_series` 필드 전체(앞부분뿐 아니라 괄호 안까지)를 `OT-검색바운더리.md`의 저널·시리즈 표와 대조할 것. 특히 단행본·편저서 수록 챕터는 시리즈명이 뒤쪽 괄호에 있는 경우가 많다. 신규 청크 완료 후 다음 스크립트류 점검을 권장:
```
python3 -c "
import csv, re
with open('worklist/download_queue.csv', encoding='utf-8') as f:
```
