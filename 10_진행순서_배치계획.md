# 10 · 진행 순서 · 배치 계획

전체 **2395건**(3주석서 미보유 union)을 빠짐없이 처리한다.
처리 순서는 **가벼운·타겟팅 가능한 것부터 → 무거운 순서로**(사용자 지정),
**사사기 핵심(인물 서사) 우선**으로 진행한다. → 데이터량 **오름차순**이 되며,
주변부인 General·ANE이 맨 마지막에 온다.

## 정렬 원칙
- **1차 (진행 순서):** 카테고리 데이터량 **오름차순** = 가벼운 것부터. 사사기 인물 서사가 앞, General·ANE이 뒤.
- **2차 (배치 내부 순서):** ① 사사기 직접관련(UNION 549 포함) → ② `OT-검색바운더리` **통독** 저널·시리즈 → ③ 표적 → ④ 기타.
- **청크 크기:** 1청크 ≈ **150건**(한 세션 처리 단위). 초과 배치는 분할.
- 첫 배치(Tola·Jair 1건)는 파이프라인 **파일럿** 겸용 — 여기서 정제·식별·태깅 포맷을 확정한다.

## 청크당 작업 정의 (파이프라인 단계 1→4)
각 청크에서 다음을 수행하고 `worklist/<청크ID>.csv`로 산출한다.

1. **정제 Clean** — xlsx 원행의 OCR 오류 교정, 서지 정규화.
   - 컬럼(18): `id, category, author, year, title_raw, title, journal_series, volume, pages, cite_Sasson, cite_Smith, cite_Gross, boundary, id_type, identifier, confidence, status, notes`
   - `title_raw`에 원행 전문을 보존하므로 저자 원형도 여기서 대조 가능(별도 author_raw 불필요).
2. **식별 Resolve** — **`40_검증방법론.md` 절차 준수**: ① bib 원본(bib_sasson/smith) title 대조 → ② 외부 DB 확정 → ③ 신뢰등급 A/B/C 부여. 실패는 `status=UNRESOLVED`.
3. **바운더리 태깅** — `OT-검색바운더리`의 저널·시리즈와 대조해 `boundary=통독/표적/off-list` 부여.
   - off-list라도 삭제하지 않고 보류(Festschrift·단행본 가능성).
4. **중복 재확인** — 드라이브 11,187개와 제목/DOI로 재대조 → 성씨+연도 오판 보정.
   - status: `RESOLVED / UNRESOLVED / HELD_ALREADY / SKIP`

> 확보(다운로드)·드라이브 반영은 별도 단계(5~6)로, **사용자 승인 후** 일괄 진행.
