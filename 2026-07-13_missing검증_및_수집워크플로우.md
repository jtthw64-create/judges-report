### 1-3. ⚠️ 품질 리스크 (수집 정확도 직결)

**(a) 매칭 방식의 한계 — 문서 스스로 인정**
> 매칭 = 성씨 + 연도. 동성·동년 우연일치로 **과대판정**(실제론 미보유인데 보유로), 표기차로 **과소판정**(실제론 보유인데 미보유로) 가능.
- → xlsx의 "미보유" 판정은 **후보**일 뿐, 확정 아님. 실제 확보 전 재확인 필요.

**(b) OCR 오류 다수 — "가짜 없이" 목적에 직접 위협**
- 자동 패턴 검출만으로 **최소 213행(~9%)** 에서 깨짐 확인. 실제론 더 많음.
- 예시:
  | xlsx 표기 | 정확한 형태(추정) |
  |---|---|
  | `Ftnsham 1961` | Fensham, F. C. |
  | `Bai, Mieke` | **Bal**, Mieke |
  | `Barthilemy` | Barthélemy |
  | `The Book ofJudges` | The Book of Judges |
  | `Insaiption` | Inscription |
  | `Kapita 14` | Kapitel 14 |
  | `Lirera(ry)` / `~tude` | Literary / Étude |
- → 저자·제목이 깨진 채로 검색하면 **오탐(다른 자료)·미검출** 발생. 정제 없이 다운로드하면 "가짜" 유입 위험.

**(c) 저널/시리즈 정보 미정규화**
- Title 필드에 저널명·권호·페이지가 뒤섞여 있음 (예: `JSOT 51, 1991, 254+`). 별도 필드 분리 안 됨.
- → 사용자 제공 예정인 **참고 저널·시리즈 바운더리**로 필터링하려면 정규화 필요.

## PART 2. 수집 워크플로우 설계 (초안)

> 목표: 미보유 후보 → **실재 검증된 서지레코드** → 확보 → 드라이브 반영.
> 가짜 방지 대전제: **모든 항목은 권위 서지 DB(CrossRef/WorldCat 등)에서 실재 확인된 것만** 확보 대상으로 승격. LLM이 서지를 지어내지 않음.

### 단계
| # | 단계 | 내용 | 산출물 (judges report 내) |
|---|---|---|---|
| 0 | **범위 확정** | 우선순위 결정: 사사기 직접관련 549 먼저 vs 전체 2395. 저널·시리즈 바운더리 적용 | `scope.md` |
| 1 | **정제 Clean** | OCR 교정 + 서지 정규화(저자·연도·제목·저널/시리즈·권호·페이지 분리) | `worklist_clean.csv` |
| 2 | **식별 Resolve** | 각 항목을 DOI/ISBN/OCLC로 해소 → **실재 검증**. 미해소 항목은 `unresolved`로 격리 | `worklist_resolved.csv` |
| 3 | **중복 재확인** | 드라이브 11,187개와 제목/DOI 기준 재대조 (성씨+연도 오판 보정) | 갱신된 status |
| 4 | **우선순위** | 인물별/주제별 + 저널·시리즈 가중치로 확보 순서 | `priority.md` |
| 5 | **확보 Acquire** | 다운로드는 **사용자가 실행**(접근권/저작권). Claude는 위치·식별자·링크까지 제시 | `acquire_queue.md` |
| 6 | **반영 Integrate** | 주제 폴더 배치안 + bib 추가안 + INDEX 갱신안 (**실제 이동/쓰기는 승인 후**) | `integration_plan.md` |
| 7 | **추적 Track** | 상태: 미착수 / 검증됨 / 확보 / 실패 / 보류. 라운드마다 갱신 | `status_tracker.csv` |

### 상태 코드 (status_tracker)
- `NEW` 미착수 · `CLEANED` 정제완료 · `RESOLVED` 실재확인 · `UNRESOLVED` 식별실패(보류) · `HELD_ALREADY` 재확인 결과 보유중 · `QUEUED` 확보대기 · `ACQUIRED` 확보완료 · `FILED` 드라이브 반영완료 · `SKIP` 범위외

### 가짜 방지 체크리스트 (매 항목)
- [ ] 저자·제목이 OCR 교정되었는가
- [ ] DOI/ISBN/OCLC 중 최소 1개로 실재 확인되었는가
- [ ] 확인된 서지가 원 인용(주석서)과 저자·연도·주제 일치하는가
- [ ] 드라이브 기보유 여부 제목 기준 재확인했는가
- [ ] 출처 없는 추정으로 채운 필드는 `[추정]` 표기했는가
