# 30 · 수집 전략 — 2트랙

최종 산출물은 두 트랙 모두 **`worklist/download_queue.csv`** (= 새로 다운받을 목록)로 합류한다.
가짜 금지: 두 트랙 모두 **실재(DOI/ISBN/서지 DB) 검증된 항목만** 다운로드 목록에 올린다.

## Track 1 — Known-missing (리스트에 있으나 파일 없음)
입력: 3주석서 미보유 union **2395건** (xlsx). → 배치 A~K (`10_진행순서_배치계획.md`).
처리: 정제 → 식별(실재검증) → 바운더리 태깅 → 중복 재확인.
**다운로드 대상 = status `RESOLVED` 이면서 `HELD_ALREADY` 아님.**
→ 해당 행을 `download_queue.csv`에 편입 (source_track=1).

## Track 2 — Discovery + Snowball (리스트에도 없음)
주석서가 인용하지 않았거나 목록에 없는 자료를, 주제어·카테고리로 **발굴**한다.
**실행 시점: Track 1 전체(2395건) 완료 후 착수** (확정). 발굴 시 이미 만들어진 Track 1 목록 전체를 중복제거 기준으로 쓸 수 있어 유리하다.

### 단계
| 단계 | 내용 |
|---|---|
| **D1 Seed 검색** | 카테고리 주제어로 웹 검색. **`OT-검색바운더리`의 저널·시리즈 안에서**, 통독>표적 순. 최신·핵심 우선. |
| **D2 검증 Verify** | 후보를 DOI/ISBN/서지 DB로 **실재 확인**. 미확인은 버리지 말고 `UNVERIFIED`로 격리. LLM 추정 서지 **금지**. |
| **D3 중복 제거** | ① 드라이브 11,187개 ② Track 1 목록 ③ 기수집 discovery 와 대조. 이미 있으면 `SKIP`. |
| **D4 Snowball** | 검증된 신자료의 **참고문헌(backward citation)** 을 추출 → **바운더리 내 항목만** 남김 → D2로 재귀. |
| **D5 편입** | 남은 신규·검증·미보유 항목을 `download_queue.csv`에 편입 (source_track=2, origin=출처자료ID). |

### Snowball 수렴 규칙 (폭증 방지)
- **깊이 2단계 확정** (Seed → 그 참고문헌 → 그 참고문헌의 참고문헌에서 중단).
- 매 단계 **바운더리 필터 + 3중 중복제거**로 걸러 수렴시킨다.
- **연도 범위 제한 없음** (고전·최신 모두 발굴, 학술 완전성 우선).
- off-list(바운더리 밖) 항목은 snowball 확장에 쓰지 않음(다운로드 목록엔 보류 기재만).
- 한 카테고리에서 신규 발굴이 연속 0건이면 종료.

## 최종 산출물: download_queue.csv
사용자가 실제로 받을 목록. **다운로드는 사용자 승인·실행** (Claude는 목록·식별자·접근경로까지).

컬럼(제안):
`id, source_track, category, author, year, title, journal_series, boundary, id_type, identifier, origin, priority, status, notes`
- `source_track`: 1(known-missing) / 2(discovery)
- `origin`: track2에서 어느 자료의 참고문헌에서 나왔는지(snowball 추적). track1은 주석서명(Sasson/Smith/Groß).
- `status`: `QUEUED`(받을 것) / `GOT`(확보완료) / `FILED`(드라이브 반영) / `HOLD`(보류)
- `priority`: 통독 > 표적 > off-list, 사사기 직접관련 가중.

## 실행 순서 (확정)
**1단계 — Track 1 전체:** 배치 A~K(진행 순서 1→11)를 모두 처리 → `download_queue.csv`(source_track=1) 완성.
**2단계 — Track 2:** Track 1 완료 후, 카테고리별로 D1~D5 발굴 → `download_queue.csv`(source_track=2) 추가.
각 단계·청크 결과는 `20_PROGRESS_TRACKER.md`에 기록.

## 확정 사항
1. 실행: **Track 1 전체 먼저 → Track 2 나중** ✅
2. Snowball 깊이: **2단계** ✅
3. Track 2 연도 범위: **제한 없음** ✅
