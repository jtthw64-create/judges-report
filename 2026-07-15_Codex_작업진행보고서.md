5. 다운로드·삭제·외부 발송·원본 폴더 쓰기는 사용자 승인 전에 하지 않는다.

### 발견한 인프라 이상

- `.git/refs/heads/main` 대신 `.git/refs/heads/main.lock`, `.git/refs/remotes/origin/main` 대신 `origin/main.lock`이 남아 있다.
- lock 파일에는 둘 다 커밋 `16f92a405e62ef43915d0920a5ef6d750f515fc4`가 기록되어 있지만, Git은 현재 `main`을 커밋이 없는 브랜치로 인식한다.
- 안전을 위해 lock 삭제·ref 복구·pull·push는 수행하지 않았다. Git 작업 전에 이 상태를 먼저 복구해야 한다.

## 2026-07-15 · 대시보드 반응형 UI 수정

### 범위 정정

- Codex의 역할은 아이템 정제가 아니라 **현황 대시보드 UI 수정**으로 한정한다.
- 이번 작업에서 데이터, 필터·검색·저장 기능, Apps Script 백엔드는 변경하지 않았다.

### 구현 내용

- 데스크톱:
  - 최대 콘텐츠 폭을 1200px에서 1680px로 확대했다.
  - 테이블의 1080px 최소폭과 가로 스크롤 의존을 제거했다.
  - 8개 열에 화면 비율 기반 너비를 배분하고, 셀 여백·글자·버튼을 밀도 있게 조정했다.
  - 재검토·교수님 코멘트 입력칸을 고정폭에서 열 너비 100%로 변경했다.
- 모바일(720px 이하):
  - 테이블 행을 아이템별 카드로 자동 전환한다.
  - 받음, 우선순위, 자료, 저널·시리즈, 접근, 원본 위치, 재검토, 교수님 코멘트를 모두 세로로 표시한다.
  - 통계는 4열 그리드, 검색창은 전체 폭, 필터는 4개 버튼 그리드로 표시한다.
  - 카테고리는 상단에서 가로 스크롤하도록 해 본문 카드 폭을 침범하지 않게 했다.

### 변경 파일

- `build_dashboard.py`
- `index.html` (자동 재생성)
- `download_dashboard.html` (자동 재생성)

- `index.html` / `download_dashboard.html`: 바이트 단위 동일.

- 현재 세션에 연결 가능한 브라우저 인스턴스가 없어 실제 뷰포트 스크린샷 기반 시각 검증은 수행하지 못했다.
